# Meridian CRM — Полное описание проекта для защиты на хакатоне

---

## Что такое Meridian CRM

**Meridian CRM** — это комплексная бизнес-платформа нового поколения, объединяющая CRM, управление задачами, каталог товаров и услуг, финансы, проектный менеджмент и AI-ассистента в единой системе. Платформа создана для команд от 5 до 500 человек и охватывает весь цикл работы с клиентами: от первого контакта до закрытой сделки и последующего проектного сопровождения.

---

## Технологический стек

| Слой | Технологии |
|---|---|
| **Frontend** | React 18, TypeScript, Vite, TanStack Query, Effector, CSS Modules, dnd-kit |
| **Backend** | Node.js, Fastify, TypeScript, InversifyJS (IoC/DI), JWT |
| **База данных** | PostgreSQL, Drizzle ORM |
| **AI** | DeepSeek API (GPT-совместимый), потоковая передача ответов |
| **Realtime** | WebSocket (собственный хаб, broadcastOrg) |
| **Инфраструктура** | Docker, Drizzle Migrations, seed-scripts |

## Разделы боковой панели (Sidebar) — детальное описание

### 1. Дашборд

**Маршрут:** `/dashboard`

**Что умеет:**
- Показывает KPI-карточки реального времени: количество лидов, сумму сделок, активных клиентов, задачи
- Отображает воронку конверсии лидов (новый → переговоры → выигран/потерян)
- Показывает последние активности CRM-команды с временными метками
- Виджет ближайших задач и дедлайнов
- Быстрый доступ к созданию нового лида, задачи, контакта прямо с дашборда

**Технические преимущества:**
- Данные подгружаются параллельно через TanStack Query с SWR-стратегией (stale-while-revalidate)
- WebSocket обновляет дашборд в реальном времени при событиях от других пользователей
- Адаптивная сетка виджетов под любой экран

---

### 2. CRM → Контакты

**Маршрут:** `/crm/contacts`

**Что умеет:**
- Полноценная база контактов с поиском, фильтрацией по статусу, сегменту, компании
- CRUD-операции: создание, просмотр карточки, редактирование, удалённое (soft) удаление
- Привязка к компаниям: один контакт — одна компания с переходом на карточку компании
- Теги: каждому контакту можно назначить произвольные теги для сегментации
- Статусы контакта (active/inactive/blocked) с цветовой индикацией
- Пагинация с мгновенным откликом через TanStack Query

**Технические преимущества:**
- `ilike`-поиск на уровне БД (PostgreSQL), а не in-memory фильтрация
- Отдельная таблица `crm_contact_tags` (многие ко многим) без потерь производительности
- Soft delete: контакт скрывается, но не теряется — история переписки и лиды сохраняются

---

### 3. CRM → Компании

**Маршрут:** `/crm/companies`

**Что умеет:**
- Карточки компаний с полями: отрасль, сайт, email, телефон, город, количество сотрудников, годовая выручка
- Статус компании и сегмент (Enterprise, SMB, Startup и т.д.)
- Просмотр всех контактов, связанных с компанией
- Быстрое создание нового контакта прямо из карточки компании
- Фильтрация по отрасли, статусу, сегменту

**Технические преимущества:**
- FK с `crm_contacts.company_id` даёт мгновенную выборку контактов компании без join-тяжёлых запросов
- Одна модель компании используется и в лидах, и в сделках — нет дублирования данных

---

### 4. CRM → Лиды

**Маршрут:** `/crm/leads`

**Что умеет:**
- Канбан-доска лидов с drag-and-drop между этапами воронки
- Этапы сделки настраиваются: создание кастомных этапов, изменение позиции, установка вероятности
- Карточка лида: сумма, валюта, приоритет, вероятность, ответственный менеджер, источник, дедлайн
- Привязка лида к контакту и компании
- Причина проигрыша при переводе лида в статус «Потерян»
- Статистика лидов: конверсия, средний чек, распределение по этапам
- История изменений лида (activity log)

**Технические преимущества:**
- `getLeadKanban` агрегирует лиды по столбцам за один запрос с join на этапы — без N+1
- Перетаскивание оптимистично обновляет UI (dnd-kit), запрос к API происходит фоново
- WebSocket-событие `lead_moved` транслируется всем участникам организации

---

### 5. Задачи → Воронки (Kanban-доски)

**Маршрут:** `/organizations/:id` → `/departments/:id` → `/departments/:id/pipelines/:pipelineId`

**Что умеет:**
- Иерархия: Организация → Отдел (Department) → Воронка (Pipeline) → Колонки → Задачи
- Перетаскивание задач между колонками с обновлением позиций (drag-and-drop через dnd-kit)
- Перетаскивание самих колонок для изменения порядка
- Полная карточка задачи: описание, дедлайн, исполнители, теги, приоритет, комментарии, вложения
- Создание воронки из готовых шаблонов: «IT-проект», «Продажи», «Баг-трекер», «HR»
- Фильтры на доске: по исполнителю, тегам, приоритету, дедлайну
- Переключение вида: Kanban ↔ Календарь
- RBAC: каждая воронка имеет политики доступа (кто может создавать/перемещать/удалять задачи)

**Технические преимущества:**
- Realtime WS: все участники видят изменения на доске мгновенно (`realtime_board_journal`)
- dnd-kit даёт 60fps анимации даже на 200+ задачах
- `position`-поле с fractional indexing — нет перерасчёта всех позиций при каждом перемещении

---

### 6. Задачи → Избранные воронки

**Маршрут:** `/favorite-pipelines`

**Что умеет:**
- Персональный список избранных воронок пользователя из всех его организаций
- Быстрый переход на нужную доску без навигации по дереву организация → отдел → воронка
- Добавление/удаление из избранного прямо с заголовка воронки

**Технические преимущества:**
- `user_favorite_pipelines` — отдельная таблица связей user↔pipeline
- Данные загружаются при старте приложения и кешируются — навигация мгновенная

### 7. Каталог → Товары

**Маршрут:** `/catalog/products`

**Что умеет:**
- Полный каталог товаров: SKU, артикул, закупочная цена, цена продажи, единица измерения
- Категории товаров с иерархией
- Управление остатками: текущий сток, минимальный остаток, склад
- Операции поступления/списания/корректировки через `stock_movements`
- Инвентаризационный отчёт по складу
- Поиск и фильтрация по категории, статусу, наличию на складе

**Технические преимущества:**
- `stock_movements` — полный audit trail склада: каждая операция фиксируется с reason и userId
- `inventory_summary` — materialized view в Drizzle, считает остатки по всем складам за один запрос
- Цена хранится в целых числах (копейки/центы) — нет проблем с floating point

### 8. Каталог → Услуги

**Маршрут:** `/catalog/services`

**Что умеет:**
- База услуг: название, описание, категория, цена, единица (час/проект/месяц), длительность
- Статус услуги (активна/архивирована)
- Использование услуг в коммерческих предложениях и сделках
- Быстрое создание и редактирование через модальную форму

**Технические преимущества:**
- Единый каталог для товаров и услуг позволяет использовать их взаимозаменяемо в строках КП/сделок (`crm_deal_line_items`)
- Таблица `services` хранит `is_active` и `deleted_at` — мягкое удаление с сохранением истории

### 9. Финансы

**Маршрут:** `/finance`

**Что умеет:**
- **Коммерческие предложения (КП):** создание, статусы (черновик/отправлено/принято/отклонено), сумма, контрагент
- **Счета на оплату (инвойсы):** создание, статусы (черновик/отправлен/оплачен/просрочен/отменён), дедлайн, оплаченная сумма
- Индикаторы просроченных платежей с предупреждающим выделением
- KPI-карточки: общая сумма КП, выставленные счета, оплачено, просрочено
- Связь инвойсов и КП с компаниями и сделками CRM

**Технические преимущества:**
- `sales_quotes` и `sales_invoices` независимы от `crm_deals` — финансовые документы живут дольше воронки
- Статусные переходы валидируются на бекенде — нельзя перевести оплаченный счёт обратно в черновик
- Таблица `crm_deal_line_items` с `productId`/`serviceId` позволяет автоматически формировать строки КП из каталога

### 10. Проекты

**Маршрут:** `/projects`

**Что умеет:**
- Создание проектов с полями: название, описание, статус, приоритет, даты начала/окончания, бюджет, цвет
- 5 статусов: Планирование, Активный, Приостановлен, Завершён, Архив
- 4 уровня приоритета: Низкий, Средний, Высокий, Критичный
- Прогресс-бар для каждого проекта (0–100%)
- KPI-лента: всего проектов, активные, завершённые, суммарный бюджет
- Поиск и фильтрация по статусу
- Управление участниками: добавление членов команды с ролями (owner/member/viewer)
- Цветовая кастомизация карточек
- Предупреждение о просроченных дедлайнах
- Карточки с акцентной полосой цвета, прогресс-баром и метаданными

**API:**

| Метод | Путь | Действие |
|---|---|---|
| `GET` | `/projects?orgId=` | Список проектов с фильтрами |
| `POST` | `/projects?orgId=` | Создать проект |
| `GET` | `/projects/:id?orgId=` | Получить проект |
| `PATCH` | `/projects/:id?orgId=` | Обновить проект |
| `DELETE` | `/projects/:id?orgId=` | Удалить проект |
| `GET` | `/projects/:id/members?orgId=` | Участники проекта |
| `POST` | `/projects/:id/members?orgId=` | Добавить участника |
| `DELETE` | `/projects/:id/members/:userId?orgId=` | Удалить участника |

**Технические преимущества:**
- Полный backend-модуль по архитектуре IoC (Repository → Service → Controller)
- Мягкое удаление (`deleted_at`) — данные не теряются
- Создание проекта автоматически добавляет создателя как `owner` в `project_members`
- Все операции логируются через `AuditService`

### 11. AI (Искусственный интеллект)

**Маршрут:** `/ai`

**Что умеет:**

**Вкладка «Чат»:**
- Полноценный AI-чат с контекстом всей CRM: лиды, контакты, компании, задачи, финансы
- Потоковая передача ответов (streaming) — AI печатает в реальном времени
- Системный промпт включает реальные данные организации: имя менеджера, список лидов, суммы сделок, статистику

**Вкладка «Инсайты»:**
- Автоматический анализ воронки продаж
- Выявление узких мест: где теряются лиды, какие этапы застревают
- Рекомендации по приоритизации работы с лидами

**Вкладка «Генерация»:**
- Генерация коммерческих предложений, писем клиентам, скриптов продаж
- Шаблоны промптов для типовых задач

**Встроенный AI Copilot:**
- Плавающая кнопка AI на всех страницах приложения
- Быстрый доступ к AI-чату без перехода на `/ai`
- Контекст автоматически строится из данных текущей страницы

**Скоринг лидов:**
- `POST /ai/score-lead/:id` — AI оценивает лид по 10-балльной шкале с объяснением
- Обогащение контакта: `POST /ai/enrich-contact/:id` — AI дополняет профиль контакта

**Технические преимущества:**
- DeepSeek API с SSE-стримингом через `streamInto` утилиту
- `buildAiContext` / `useAiContext` — умный сборщик контекста: сканирует React Query кеш, агрегирует данные из всех загруженных страниц
- Контекст включает реальные данные лидов: стадия, сумма, ответственный, дата закрытия
- AI видит статистику лидов даже без явной навигации пользователя на страницу лидов

### 12. Отчёты

**Маршрут:** `/reports`

**Что умеет:**
- Сводный отчёт по продажам за период (7д/30д/90д/всё время)
- Воронка конверсии: количество и сумма лидов на каждом этапе, процент конверсии между этапами

- Тренды: график новых лидов и выигранной выручки по датам

**Технические преимущества:**

- Все агрегации выполняются на уровне PostgreSQL (GROUP BY, SUM, COUNT) — минимальная нагрузка на Node.js
- Данные кешируются в TanStack Query с инвалидацией при изменении лидов

## Ключевые технические преимущества

### 1. Инверсия зависимостей (IoC/DI)
Весь backend построен на InversifyJS: каждый модуль (Controller → Service → Repository) инжектируется через символы, что делает систему полностью тестируемой и расширяемой.

### 2. Realtime WebSocket
Собственный WS-хаб (`wsHub.broadcastOrg`) транслирует события (движение задачи, создание лида, новый комментарий) всем участникам организации без polling.

### 3. Умный React Query кеш
TanStack Query используется не только для загрузки данных, но и как локальный стейт-менеджер. AI-контекст строится путём сканирования кеша (`queryClient.getQueriesData`), что позволяет AI видеть все данные, которые пользователь уже загрузил.

### 4. Мягкое удаление везде
Все критичные сущности (контакты, компании, лиды, проекты, задачи) используют `deleted_at` — данные никогда не теряются, возможен аудит и восстановление.

### 5. CSS Modules + Design Tokens
Весь UI построен на CSS custom properties (`--color-accent`, `--color-bg-secondary`, etc.) — темизация работает без JavaScript, переключение тем мгновенное.

### 6. Fractional Positioning
Задачи и колонки используют `position`-поле для сортировки. При перетаскивании обновляется только позиция перемещённого элемента, а не всего списка.

## Демо-сценарий для судей

### Шаг 1: Дашборд (30 сек)
Откройте `/dashboard`. Покажите KPI-карточки с реальными данными из seed. Укажите на воронку конверсии.

### Шаг 2: CRM → Лиды (1 мин)
Перейдите на `/crm/leads`. Откройте kanban-вид. Перетащите лид из «Переговоры» в «Выигран» — система запросит причину. Откройте карточку лида, покажите историю изменений.

### Шаг 3: Kanban-доска задач (1 мин)
Зайдите в любой отдел → воронку. Перетащите задачу между колонками. Откройте карточку задачи: комментарии, дедлайн, вложения. Включите вид «Календарь».

### Шаг 4: AI (1 мин)
Перейдите на `/ai`. Задайте вопрос: *«Какие лиды требуют срочного внимания?»* — AI ответит с конкретными именами лидов и суммами. Затем спросите: *«Напиши КП для Сбер на внедрение CRM»*.

### Шаг 5: Проекты (45 сек)
Откройте `/projects`. Покажите KPI-ленту. Создайте новый проект. Задайте дедлайн в прошлом — появится индикатор просрочки.

### Шаг 6: Финансы (30 сек)
Откройте `/finance`. Покажите счета со статусами. Обратите внимание на просроченные счета с предупреждением.

### Шаг 7: Отчёты (30 сек)
Откройте `/reports`. Переключите период с 30д на 90д. Покажите рейтинг менеджеров и воронку конверсии.
